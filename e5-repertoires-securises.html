Situation professionnelle numéro 1

Création et sécurisation de répertoires réseau via script et serveur bastion

Description :

La création de répertoires sécurisés permet à une entreprise de disposer de ses dossiers partagés dans un environnement contrôlé et fiable. Afin d'industrialiser les demandes et d'éviter les erreurs de configuration manuelle, le processus s'appuie sur un fichier de calcul (Excel) générant les paramètres de sécurité, appliqué ensuite via un script sécurisé depuis un serveur bastion.

Le cahier des charges

L'expression des besoins

Notre société souhaite normaliser et sécuriser la création des répertoires partagés pour ses collaborateurs. L'objectif est d'attribuer précisément les droits d'accès (qui est propriétaire, qui a les droits de lecture/écriture) selon les demandes, tout en garantissant la traçabilité des opérations via un système de ticketing.

La description de l'existant

Actuellement, les demandes sont centralisées dans un "backlog" (gestionnaire de tickets). Les dossiers de partages existent sur un serveur de fichiers central, mais l'accès direct à ce serveur est restreint pour des raisons de sécurité évidentes.

L'analyse des choix

Pour réaliser cette tâche en toute sécurité et de manière industrielle :

  • Le serveur Bastion : Choisi pour isoler le réseau d'administration. C'est le seul point d'entrée pour exécuter des commandes d'administration sur les serveurs de fichiers.
  • Le fichier Excel "moulinette" : Permet de traduire les besoins humains (noms, chemins) en code ou paramètres systèmes stricts et sans fautes de frappe pour le script.

Mise en œuvre

Prise en charge et analyse du ticket

Le processus débute par la vérification du backlog de l'entreprise :

  1. Parcours des tickets en attente de traitement.
  2. Auto-attribution du ticket de création de répertoire pour en devenir le responsable.
  3. Analyse des informations cruciales fournies par le demandeur :
  • Le chemin cible de l'arborescence où le dossier doit être créé.
  • L'identité du demandeur.
  • L'identité du futur gestionnaire du dossier.
  • Les éventuelles restrictions ou demandes particulières.

Préparation des données et génération de la configuration

Pour éviter les erreurs d'exécution, un tableau Excel est utilisé comme traducteur de paramètres :

  1. Saisie des informations extraites du ticket dans les colonnes correspondantes (Chemin, Gestionnaire, etc.).
  2. Le tableau Excel calcule automatiquement et génère une ligne de commande ou une chaîne de caractères spécifique.
  3. Cette ligne représente la future configuration de sécurité et les droits ACL (Access Control List) du dossier.
  4. Copie de cette ligne générée dans le presse-papier.

Connexion au serveur bastion et exécution du script

L'action ne pouvant pas être faite depuis le poste client local pour des raisons de sécurité :

  1. Connexion sécurisée au serveur Bastion de l'entreprise.
  2. Ouverture du terminal et appel du script d'automatisation de création de dossiers.
  3. Collage de la ligne de sécurité générée précédemment par le fichier Excel.
  4. Exécution du script : le répertoire est créé sur le serveur de fichiers cible avec les bons privilèges appliqués.

Clôture et vérification auprès du collaborateur

Afin de finaliser le traitement :

  1. Demande au collaborateur de redémarrer sa machine (nécessaire pour actualiser ses jetons de sécurité Active Directory et monter le nouveau dossier).
  2. Validation du bon accès avec le collaborateur.
  3. Clôture définitive du ticket dans le backlog.
