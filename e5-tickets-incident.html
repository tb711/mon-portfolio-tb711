Situation professionnelle numéro 2

Résolution d'un incident d'environnement de développement et migration d'OS VM

Description :

Le maintien en condition opérationnelle des outils de développement est crucial pour les équipes de l'entreprise. Suite à un dysfonctionnement bloquant de l'application Visual Studio Code sur une machine virtuelle Windows Server 2016, un diagnostic a mis en évidence une incompatibilité de version d'OS. L'incident a nécessité une solution temporaire puis une demande de migration de la VM vers Windows Server 2019 pour répondre au besoin du collaborateur.

Le cahier des charges

L'expression des besoins

Un collaborateur de l'entreprise est bloqué dans son travail quotidien : lorsqu'il tente d'exécuter son code depuis l'IDE Visual Studio Code, le terminal intégré ne s'affiche pas et génère un message d'erreur. L'objectif est de rétablir un environnement de développement pleinement fonctionnel avec la version la plus récente possible de l'application.

La description de l'existant

Le collaborateur travaille à distance sur une machine virtuelle (VM) hébergée sur l'infrastructure de l'entreprise. Le système d'exploitation de cette VM est Windows Server 2016.

L'analyse des choix

Lors de la résolution de cet incident, plusieurs étapes logiques d'analyse ont été menées :

  • La réinstallation propre : Recommandée par les outils internes pour éliminer un problème de corruption de fichiers.
  • Le "Downgrade" (Rétrogradation de version) : Choisi comme solution de secours immédiate pour débloquer le collaborateur.
  • La migration de l'OS de la VM : Solution pérenne retenue pour permettre l'exécution de la dernière version logicielle de VS Code, exigeant des prérequis système plus récents.

Mise en œuvre

Prise en charge de l'incident et premiers diagnostics

  1. Consultation du backlog de l'entreprise et auto-attribution du ticket d'incident.
  2. Prise de contact avec le collaborateur pour analyser le comportement anormal de Visual Studio Code (absence d'affichage du terminal d'exécution).
  3. Sollicitation du Chatbot interne de l'entreprise pour vérifier si cet incident est déjà connu.
  4. Application de la consigne du Chatbot : désinstallation complète de VS Code et réinstallation propre de la toute dernière version disponible.
  5. Constat d'échec : le problème persiste malgré la réinstallation.

Recherche de solutions et contournement temporaire

  1. Élargissement de la recherche technique sur Internet avec les codes d'erreurs relevés.
  2. Identification de la cause racine : la version de Windows Server 2016 de la VM est devenue trop ancienne et n'est plus totalement supportée par les dernières briques d'exécution de la mise à jour récente de VS Code.
  3. Solution de contournement : installation d'une version antérieure de VS Code compatible avec Windows Server 2016. Le terminal fonctionne à nouveau.

Demande de migration d'infrastructure

Le collaborateur ayant exprimé le besoin absolu d'utiliser la dernière version de VS Code pour ses projets :

  1. Formulation d'une demande d'escalade / changement pour migrer la machine virtuelle du collaborateur vers un système d'exploitation plus récent (Windows Server 2019).
  2. Traitement de la migration par l'équipe d'infrastructure dédiée (action réalisée en asynchrone sans ma participation directe).

Recette finale et clôture du ticket

  1. Une fois la VM migrée avec succès sous Windows Server 2019 par l'équipe infra, reprise de contact avec le collaborateur.
  2. Procédure d'installation propre de la dernière version de Visual Studio Code sur le nouvel OS.
  3. Phase de tests : ouverture de l'IDE, lancement du terminal et exécution de code. Tout est parfaitement fonctionnel.
  4. Validation de la satisfaction du collaborateur et clôture du ticket d'incident dans le backlog.
